Interacción Humano-Ordenador

Ahora que ya hemos visto cómo funciona las interfaces y cómo hacerlas mas efectivas pasaremos a
desarrollar el tema de la interacción Humano – ordenador, donde debemos tener presente dos conceptos, modelo mental y modelo conceptual:
-Modelo mental:
Este es no es ni completo, ni consistente, ni exacto y esto por la sencilla razón de que se constituye de la abstracción del conocimiento interno del usuario que tiene del sistema
cuando lo está utilizando.

transferencia.tec.mx


-Modelo conceptual:
Este es muy contrario al anterior ya que tiene una información que describe, con diagramas y
notaciones más o menos formales, el conocimiento que debe tener una persona sobre el sistema.
Así pues, las interfaces hacen posible que la interacción entre el usuario y el ordenador como si de
un diálogo se tratara. Desde el punto de vista del usuario:
-El modelo conceptual representa la información que cualquier usuario debería tener o adquirir sobre
el sistema.
-Está formado por un conjunto de elemento (conceptos) y relaciones entre los mismo observables desde
el exterior.

Este modelo debe garantizar que le dará la debida información al usuario sobre lo que hace el
sistema y los mecanismos para llevarlo a cabo, esto con el fin de favorecer el aprendizaje del
sistema, para esto se sugiere que debe ser creado según estos principios:
-Asimilable, mediante el uso de conceptos familiares.
-Consistente, coherentes y bien formulado.
-Simple, uso de descripciones comprensibles para un usuario medio.
Tener en cuenta que cuando se crea algún sistema se debe tener en cuenta que no solo debe ser
eficiente o simplemente funcionar, sino también deber ser comprensible.
